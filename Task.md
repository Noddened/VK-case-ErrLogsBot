Компании регулярно обращаются к логам приложений и сервисов, чтобы улучшить их, найти и исправить ошибки, защитить от уязвимостей. Небольшие стартапы в начале деятельности отслеживают информацию в ручном режиме, более крупные организации используют автоматизированные сервисы, чтобы снизить нагрузку на сотрудников поддержки.

Вам предлагается разработать бота, который сможет читать логи, собирать только логи, заданные регулярными выражениями в файле конфигурации, игнорировать логи без заданных регулярных выражений и отправлять результаты в читаемом виде. Например, бот должен уметь видеть ошибки в логах и направлять информацию о них.

### Обязательные требования к боту:
1. Разработан и реализован с помощью открытых общедоступных сервисов.

2. Способен анализировать логи из файла logs.log (файл будет сгенерирован при запуске кода).

3. Отправляет только логи, указанные в файле конфигурации, пропускает остальные.

4. Работает без сбоев, не допускает фальсификации или искажения данных.

### Алгоритм выполнения проекта:

Изучите задание и дополнительные материалы. Обратите внимание на формат представления результата.

Определите цель и задачи проекта, опишите его.

Продумайте желаемый образ результата.

Выберите оптимальные способы и средства достижения результата.

Изучите документацию по созданию ботов (в зависимости от выбора площадки для его размещения).

Скачайте приложенный к заданию код.

Скачайте компилятор языка Go.

Запустите код с помощью команды go run task.go.

Изучите файл логов logs.log.

Разработайте чат-бота.

Протестируйте работоспособность чат-бота.

Прикрепите результат (ссылку на работающего чат-бота) в форму обратной связи на обучающей платформе.

Дождитесь оценки и подведения итогов.

### Какой результат хотим получить:

Бот, который из генерируемого потока логов вычленяет логи с ошибками и отправляет их в отдельный чат.
